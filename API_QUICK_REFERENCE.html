📡 Format de Subscription

{
  "action": "subscribe",        // ou "unsubscribe"
  "data_type": "best_touch",    // best_touch | trade | kline | ewma
  "symbol": "BTCUSDT",          // Paire de trading
  "exchange": "all",            // all | binance | okx
  "interval": "1m"              // Requis pour kline: 1s | 10s | 1m | 5m
}

📊 Types de Données

Type Description Exchange Filter Interval
best_touch Best bid/ask cross-exchange avec détection d'arbitrage ✅ Oui ❌ Non
trade Trades individuels des exchanges ✅ Oui ❌ Non
kline Chandelles OHLCV agrégées ✅ Oui ✅ Requis
ewma Moyenne mobile exponentielle pondérée ✅ Oui ❌ Non

💡 Exemples d'Utilisation

JavaScript/Browser

const ws = new WebSocket('ws://localhost:8000/ws');

ws.onopen = () => {
  // S'abonner au meilleur bid/ask pour BTCUSDT
  ws.send(JSON.stringify({
    action: "subscribe",
    data_type: "best_touch",
    symbol: "BTCUSDT",
    exchange: "all"
  }));
  
  // S'abonner aux klines 1m pour ETHUSDT
  ws.send(JSON.stringify({
    action: "subscribe",
    data_type: "kline",
    symbol: "ETHUSDT",
    exchange: "binance",
    interval: "1m"
  }));
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log(message.type, message.data);
  
  // Détecter arbitrage
  if (message.type === 'best_touch' && message.data.spread < 0) {
    console.log('⚡ ARBITRAGE OPPORTUNITY:', message.data);
  }
};

Python

import asyncio
import json
import websockets

async def subscribe_to_market_data():
    uri = "ws://localhost:8000/ws"
    
    async with websockets.connect(uri) as websocket:
        # Subscription
        await websocket.send(json.dumps({
            "action": "subscribe",
            "data_type": "best_touch",
            "symbol": "BTCUSDT",
            "exchange": "all"
        }))
        
        # Recevoir les messages
        async for message in websocket:
            data = json.loads(message)
            print(f"{data['type']}: {data['data']}")

asyncio.run(subscribe_to_market_data())

⚡ Fonctionnalités Clés

  • Détection d'arbitrage automatique : Spread négatif = opportunité d'arbitrage cross-exchange
  • Formatage intelligent des prix : Précision adaptative selon la valeur de l'asset
  • Multi-exchange : Agrégation de données de Binance et OKX en temps réel
  • Multi-interval : Klines disponibles en 1s, 10s, 1m et 5m
  • Subscriptions flexibles : Filtrage par exchange (all/binance/okx)
